# Simple Geometry — план разработки

## Стек

### Генератор сайта: **Astro**

Почему Astro, а не Hugo/Jekyll/VitePress:
- **Content Collections** — кладёшь `.md` файл в папку, он автоматически становится страницей, с валидацией frontmatter.
- **MDX** — в будущем можно вставлять React/Vue/vanilla-JS компоненты прямо внутри статьи (2D/3D вьюверы).
- Официальная поддержка деплоя на **GitHub Pages** (один yml-файл).
- KaTeX через стандартные remark/rehype плагины.
- Нет lock-in на фреймворк для UI — можно добавить Three.js, D3.js, custom Canvas позже.

### Математика: **KaTeX** (через `remark-math` + `rehype-katex`)
- Быстрее MathJax, рендерится при сборке (SSG), не требует JS в браузере.
- Inline: `$...$`, block: `$$...$$` — то же самое, что в Obsidian.

### Стили: **Custom CSS (без фреймворка)**
- Шрифт: **EB Garamond** — LaTeX-подобный вид.
- Фон: `#fffff8` — классический кремово-желтоватый цвет LaTeX-документа.
- Типографика в стиле Tufte (широкие поля, сноски, чистые заголовки).
- Никакого Bootstrap/Tailwind — просто аккуратный CSS.

### Будущие 2D/3D вьюверы
- **Three.js** для 3D сцен (WebGL).
- **D3.js** или голый Canvas API для 2D интерактивной геометрии.
- Подключаются как Astro-компоненты внутри `.mdx` файлов — ничего менять в архитектуре не придётся.

### URL-структура
```
/Simple_Geometry/en/              → главная (английский, дефолт)
/Simple_Geometry/ru/              → главная (русский)
/Simple_Geometry/en/articles/     → список статей (en)
/Simple_Geometry/ru/articles/     → список статей (ru)
/Simple_Geometry/en/articles/My-Article
/Simple_Geometry/ru/articles/Моя-статья
```

### Переключатель языка
- Хранится в `localStorage` под ключом `lang` (`'en'` | `'ru'`).
- Корневой `index.astro` содержит JS-скрипт, который при первом визите читает `localStorage` и делает `location.replace` на `/en/` или `/ru/`.
- Header показывает кнопки **EN / RU**; активный язык выделен. При клике меняет `localStorage` и переходит на эквивалентную страницу другого языка.
- Если у статьи нет перевода — кнопка другого языка скрыта или задизейблена.

### Поле id в статьях
- Каждая статья в Obsidian содержит поле `id:` в frontmatter — латинский slug (строчные буквы, цифры, дефисы).
- При импорте файл переименовывается: `Моя-статья.md` → `<id>.md`.
- Оба файла `ru/<id>.md` и `en/<id>.md` имеют одинаковое имя → маппинг тривиален.
- Уникальность `id` проверяется при генерации: Claude предлагает, инструмент сверяет с существующими файлами.

### Кеш id в инструменте импорта
- При нажатии «Обновить» инструмент обходит все obsidian-файлы и читает frontmatter.
- Строится словарь `_id_map: {id → obsidian_rel_path}`.
- `get_all_articles()` принимает этот словарь и не читает файлы повторно.
- Определение статуса:
  - `new` — нет `id` в frontmatter, или `ru/<id>.md` не существует.
  - `imported` — `ru/<id>.md` существует.
  - `site_only` — файл в `ru/` чей stem не встречается ни в одном `_id_map`.
- Кортеж в `_all_articles`: `(display_path, status, id_or_None)`.

## Workflow импорта статей (с переводом)

Пайплайн инструмента импорта (шаги, которые можно включать галочками):

| # | Группа | Шаг | Авто? |
|---|--------|-----|-------|
| 0 | Предобработка | Убрать фон картинок | да |
| 1 | Предобработка | Заменить е → ё | Claude CLI |
| 2 | Анализ | Анализ текста | Claude CLI |
| 3 | Анализ | Анализ картинок | Claude CLI |
| 2 | Предобработка | Заполнить id | Claude CLI |
| 3 | Предобработка | Заполнить title | редактируемое поле |
| 4 | Анализ | Анализ текста | Claude CLI |
| 5 | Анализ | Анализ картинок | Claude CLI |
| 6 | Перевод | Перевести на английский | Claude CLI |
| 7 | Копирование | Копирование файлов | да |

**Шаг 2 — id:**
- Если `id` уже есть в frontmatter Obsidian-файла — показывается, шаг можно пропустить.
- Если нет — Claude генерирует Latin slug на основе названия файла.
- Проверка уникальности: сравнивается с `_id_map` и файлами в `ru/`.
- Результат показывается в редактируемом поле. «Применить →» записывает `id:` в Obsidian-файл.

**Шаг 3 — title:**
- Поле предзаполняется из имени файла Obsidian (дефисы → пробелы).
- Пользователь редактирует вручную.
- «Применить →» записывает `title:` в Obsidian-файл.

**Шаг 6 — Перевод:**
- Claude CLI переводит статью на английский, включая title.
- Результат отображается в редактируемом текстовом поле.
- Пользователь может исправить перевод вручную.
- «Применить →» сохраняет в `src/content/articles/en/<id>.md`.
- «Пропустить» — английская версия не создаётся, статья будет только на русском.

**Шаг 7 — Копирование:**
- Русский оригинал копируется в `src/content/articles/ru/<id>.md` (переименовывается).
- Английский вариант уже сохранён на шаге 6 (если не пропущен).
- Картинки копируются в `public/images/` (общие для обоих языков).
