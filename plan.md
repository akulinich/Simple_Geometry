# Simple Geometry — план разработки

## Стек

### Генератор сайта: **Astro**

Почему Astro, а не Hugo/Jekyll/VitePress:
- **Content Collections** — кладёшь `.md` файл в папку, он автоматически становится страницей, с валидацией frontmatter.
- **MDX** — в будущем можно вставлять React/Vue/vanilla-JS компоненты прямо внутри статьи (2D/3D вьюверы).
- Официальная поддержка деплоя на **GitHub Pages** (один yml-файл).
- KaTeX через стандартные remark/rehype плагины.
- Нет lock-in на фреймворк для UI — можно добавить Three.js, D3.js, custom Canvas позже.

### Математика: **KaTeX** (через `remark-math` + `rehype-katex`)
- Быстрее MathJax, рендерится при сборке (SSG), не требует JS в браузере.
- Inline: `$...$`, block: `$$...$$` — то же самое, что в Obsidian.

### Стили: **Custom CSS (без фреймворка)**
- Шрифт: **Computer Modern / Latin Modern** (веб-версия `lm-math` или `cmu-serif`) — настоящий LaTeX-вид.
- Фон: `#fffff8` — классический кремово-желтоватый цвет LaTeX-документа.
- Типографика в стиле Tufte (широкие поля, сноски, чистые заголовки).
- Никакого Bootstrap/Tailwind — просто аккуратный CSS.

### Будущие 2D/3D вьюверы
- **Three.js** для 3D сцен (WebGL).
- **D3.js** или голый Canvas API для 2D интерактивной геометрии.
- Подключаются как Astro-компоненты внутри `.mdx` файлов — ничего менять в архитектуре не придётся.

## Разделы сайта

### 1. Главная (`/`)
- Краткое введение / цитата.
- Список последних статей.
- Навигация.

### 2. Статьи (`/articles`)
- Список всех статей с тегами и датой.
- Фильтрация по тегам (опционально, позже).

### 3. Отдельная статья (`/articles/slug`)
- Рендер Markdown + KaTeX.
- Поддержка картинок и GIF.
- Заголовок, дата, теги.

### 4. О авторе (`/about`)
- Короткое описание.

## Workflow с Obsidian

1. Пишешь заметку в Obsidian, добавляешь frontmatter:
   ```yaml
   ---
   title: "Теорема Пифагора"
   date: 2026-04-24
   tags: [евклидова геометрия, теоремы]
   description: "Доказательство через подобие треугольников"
   ---
   ```
2. Копируешь `.md` файл в `src/content/articles/`.
3. Картинки кладёшь в `public/images/`.
4. Astro автоматически подхватывает файл при следующей сборке.

Obsidian-синтаксис конвертируется автоматически при сборке двумя remark-плагинами:

- `remark-wiki-link` (npm) — обрабатывает `[[Название статьи]]` и `[[Название|текст]]`.
- Кастомный плагин `remark-obsidian-image` — обрабатывает `![[image.png]]` → `![image.png](/images/image.png)`.

Исходники Obsidian трогать не нужно — конвертация на этапе сборки.

## Деплой на GitHub Pages

- GitHub Actions: при пуше в `main` — сборка Astro → деплой в ветку `gh-pages`.
- Файл `.github/workflows/deploy.yml` — стандартный, Astro его генерирует.
